## 建議工作流程（文件版控）

本資料夾未使用 Git；若以 OneDrive/SharePoint 協作為主，建議：

1. **更新即另存新檔**：避免覆蓋原檔，於檔名加上日期或版號（例如：`YYYY.MM.DD` 或 `YYYYMMDD`）。
2. **維持單一真實來源**：若同檔案需多方修訂，先指定「主檔」與「彙整人」，其餘以批註/修訂追蹤回傳。
3. **用修訂模式**：Word 建議開啟「修訂/追蹤修訂」、Excel 建議以註解標示修改點，減少版本歧義。
4. **重大變更留紀錄**：建議集中記在 `L/CHANGELOG.md`（或另做版本紀錄表），記錄日期、改動人、改動摘要。
